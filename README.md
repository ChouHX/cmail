# Cloudflare Email Pickup

This replaces the previous Mailu-based pickup deployment.

## Runtime

- App: Flask + SQLite
- Cache/queue: Redis
- Reverse proxy and HTTPS: 1Panel
- Public admin: `https://mail.bbbnn.me/admin/login`
- Ingest endpoint: `https://mail.bbbnn.me/api/cloudflare-email`

The application is published only on `127.0.0.1:8000`. It is not directly
reachable from the Internet; 1Panel should be the only public entry point.

## Deploy

Copy the environment template before starting the service. At minimum, use
strong random values for `ADMIN_PASSWORD`, `FLASK_SECRET`, and `INGEST_SECRET`,
and set the public URL:

```sh
cp .env.example .env
nano .env
```

Start or update the containers:

```sh
cd /opt/pickup-worker-mail
docker compose up -d --build --remove-orphans
docker compose ps
```

To use another local port, set `APP_PORT` in `.env` (for example,
`APP_PORT=18000`) and use that same port in 1Panel.

## 1Panel website setup

1. In **Websites**, create a **Reverse Proxy** website for `mail.bbbnn.me`.
2. Set the proxy address to `http://127.0.0.1:8000` (or your `APP_PORT`).
3. Enable HTTPS, request/select the certificate, and enable HTTP-to-HTTPS redirect.
4. Keep the standard forwarded headers generated by 1Panel, especially
   `Host`, `X-Real-IP`, `X-Forwarded-For`, and `X-Forwarded-Proto`.
5. If Cloudflare proxying is enabled, set SSL/TLS mode to **Full (strict)**.

After deployment, verify locally and publicly:

```sh
curl http://127.0.0.1:8000/healthz
curl https://mail.bbbnn.me/healthz
```

## Cloudflare setup

1. Put `bbbnn.me` on Cloudflare DNS.
2. Enable Email Routing.
3. Create a Worker from `worker/worker.js`.
4. Set Worker variable:
   - `INGEST_URL=https://mail.bbbnn.me/api/cloudflare-email`
5. Set Worker secret:
   - `INGEST_SECRET` from `.env`
6. In Email Routing, configure a catch-all route to this Worker.
